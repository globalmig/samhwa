## 구현 기능 목록

- 통합 대시보드 — 연도별/과제별/기관별 수수료, 미청구, 미수금, 정산 현황 종합 조회
- 기업 관리 — 기업 기본정보, 담당자 정보
- 연차별 기업분류 관리 — 과제별·연차별 기업분류, 기업분류 변경 이력
- 수행기관 관리 — 주관기관/참여기관, 기관별 사업비/수수료/정산내역
- 과제 및 연차 관리 — 과제번호, 과제명, 사업연도, 연차, 과제유형, 전문기관, 정산구분, 사업비
- 수수료 정책 엔진 — 기본/가산 수수료, 기관 수 기준 가산 규칙, 사업비 구간별 규칙, 과제유형별/정산구분별/기업분류별 계산식, 연차별 청구비율 관리, 정산면제 규칙, 예외 규칙
- 연차별 수수료 관리 — 사업비, 표준수수료, 신청수수료, 청구수수료, 연도별 미청구수수료, 과거 미청구액, 누적수수료, 당해 최종청구액, 미수금
- 미청구액 관리 — 연도별 미청구액, 과거 미청구액 이월, 누적 미청구액, 최종 청구액 자동 계산
- 미수금 및 채권 관리 — 청구/입금/미수금, 부분 수금 처리, 채권 잔액, 장기 미수금, 수금 이력
- 세금계산서 관리 — 발행, 이력 관리, PDF 출력, 템플릿 커스터마이징
- 기관 정산 관리 — 과제별 정산금(+기관별 추가금)/수수료, 정산 예정금, 지급 완료, 정산 이력
- 이메일 발송 관리 — 수수료 산출 내역 발송, 기관별/담당자별 일괄 발송, 발송 이력 관리
- 정책 변경 이력 관리 — 정책 이력, 과거 데이터 보존, 신규 데이터 자동 적용
- 권한 관리 — 시스템 관리자/회계 담당자/정산 담당자/일반 사용자, 메뉴/기능/데이터 접근 권한, 감사 로그

## 도메인 구조 분석

### 핵심 개념 관계도

```
기업(Company)
  └─ 연차별 기업분류 (분류 이력 포함)

과제(Project)
  └─ 연차(ProjectTerm) [1:N]
       └─ 수행기관(Institution) [N:M]
            ├─ 사업비 → 수수료 계산
            ├─ 미청구액 → 이월/누적
            ├─ 미수금/채권
            └─ 정산

수수료 정책 엔진 (Rule Engine)
  ├─ 기관 수 기준 가산 규칙
  ├─ 사업비 구간별 규칙
  ├─ 과제유형별 / 정산구분별 / 기업분류별 계산식
  ├─ 연차별 청구비율
  ├─ 정산면제 규칙
  └─ 예외 규칙
```

### 수수료 계산 흐름

```
사업비
  → 수수료 정책 엔진 적용
  → 표준수수료 산출
  → 신청수수료 입력 (실제 신청값)
  → 청구수수료 확정
  → 연도별 미청구수수료 = 청구수수료 - 실청구액
  → 과거 미청구액 이월 합산
  → 누적수수료 / 당해 최종청구액 자동계산
  → 미수금 추적
```

## 기능별 복잡도 및 의존성

### 1단계 — 마스터 데이터 (기반)

| 기능 | 핵심 데이터 | 복잡도 |
|------|------------|--------|
| 기업 관리 | 기업명, 사업자번호, 담당자 | 낮음 |
| 과제 및 연차 관리 | 과제번호/명, 사업연도, 연차, 유형, 전문기관, 정산구분, 사업비 | 중간 |
| 수행기관 관리 | 주관/참여기관, 기관별 사업비·수수료·정산내역 | 중간 |
| 연차별 기업분류 관리 | 분류값, 변경 이력 (시점 스냅샷 필수) | 중간 |

### 2단계 — 정책 엔진 (핵심 비즈니스 로직)

| 기능 | 특이사항 | 복잡도 |
|------|---------|--------|
| 수수료 정책 엔진 | 다중 규칙 조합 계산, 우선순위 처리, 예외 규칙 | **높음** |
| 정책 변경 이력 관리 | 정책 버전 관리 — 과거 데이터는 구버전 정책으로 보존 | **높음** |

> 이 엔진이 전체 시스템의 핵심. 잘못 설계되면 수수료 계산 전체가 틀어짐.

### 3단계 — 수수료 계산 흐름 (의존: 1+2단계)

| 기능 | 복잡도 |
|------|--------|
| 연차별 수수료 관리 | 높음 (7개 금액 필드 간 연산) |
| 미청구액 관리 | 높음 (연도 간 이월 로직) |
| 미수금 및 채권 관리 | 중간 (부분 수금, 채권 잔액 추적) |

### 4단계 — 출력 및 운영 기능

| 기능 | 외부 의존성 | 복잡도 |
|------|------------|--------|
| 세금계산서 관리 | PDF 라이브러리 (pdfkit / puppeteer) | 중간 |
| 이메일 발송 관리 | 이메일 서비스 (Nodemailer / Resend) | 중간 |
| 기관 정산 관리 | 수수료 계산 완료 후 가능 | 중간 |
| 통합 대시보드 | 전 모듈 완성 후 집계 | 중간 |

### 5단계 — 시스템 관리

| 기능 | 복잡도 |
|------|--------|
| 권한 관리 (RBAC) | 4개 역할 × 메뉴/기능/데이터 접근 제어 | 중간~높음 |
| 감사 로그 | 모든 변경 이벤트 기록 | 중간 |

## 예상 라우트 구조 (App Router)

```
app/
  (auth)/
    login/
  (main)/
    layout.tsx              ← 사이드바 + 헤더 공통 레이아웃
    page.tsx                ← 통합 대시보드
    companies/              ← 기업 관리
    company-class/          ← 연차별 기업분류 관리
    institutions/           ← 수행기관 관리
    projects/               ← 과제 및 연차 관리
    fee-policy/             ← 수수료 정책 엔진
    fees/                   ← 연차별 수수료 관리
    unclaimed/              ← 미청구액 관리
    receivables/            ← 미수금 및 채권 관리
    tax-invoices/           ← 세금계산서 관리
    settlements/            ← 기관 정산 관리
    emails/                 ← 이메일 발송 관리
    policy-history/         ← 정책 변경 이력 관리
    admin/
      users/                ← 권한 관리
```

## 개발 권장 순서

1. **환경 설정** — DB, ORM, 인증, UI 라이브러리 세팅
2. **수수료 정책 엔진** — 전체 시스템의 핵심, 먼저 설계 확정
3. **마스터 데이터** — 기업 → 과제/연차 → 수행기관
4. **수수료 계산 흐름** — 표준수수료부터 미수금까지
5. **출력/운영** — 세금계산서, 이메일, 정산
6. **대시보드 + 권한** — 마지막에 통합
